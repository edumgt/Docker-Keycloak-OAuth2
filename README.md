# Docker-Keycloak-OAuth2

1. 통합 인증 서버 (IdP) :
- Keycloak 기반 인증 서버 구축
- OAuth2 / OpenID Connect 기반 SSO
- SNS 로그인 연동 (구글, 카카오, 라인 등)
- Access Token / Refresh Token 기반 인증
2. 회원 관리 :
- 단일 사용자 식별자(Customer ID) 체계
- 회원 단계 개념 지원(예: 방문자 → 상담고객 → 회원 → 서비스 이용자 → 의료 서비스 이용자)
- 계정 상태 관리 (활성 / 휴면 / 탈퇴)
3. 동의 관리 구조 :
- 공통 동의 + 서비스별 추가 동의 구조
- 필수 / 선택 동의 구분
- 목적별 동의 분리 : 회원관리/상담·예약/커머스/마케팅/의료 서비스(외부 시스템 연계)
※ 의료정보는 외부 의료 시스템(EMR 등)에서만 처리되며, 통합 인증 시스템에는 의료정보를 저장하지 않음

보안 및 법·정책 고려사항 :
- 의료 정보와 비의료 정보 완전 분리
- 최소 정보 수집 원칙
- 글로벌 서비스 대응을 위한
- 국가/언어별 동의 버전 관리
- 개인정보 국외 이전 동의 구조
- 인증/세션 보안 정책 설계
- 비밀번호 정책
- OTP / MFA 확장 가능 구조

주요 사용자 흐름 (요약) :
- 마케팅/서비스 유입 (비회원)
- 상담 또는 예약 시도
- 필수 단계에서 통합ID 회원 전환
- SSO 기반으로 각 서비스 이용
- 특정 서비스 이용 시 추가 동의 수집
- 의료 서비스 이용 시 외부 시스템에서 별도 처리

기술 스택 (협의) :
- IdP: Keycloak
- Auth Protocol: OAuth2, OpenID Connect
- DB: PostgreSQL (Keycloak Backend)
- Frontend 연동: Web / Mobile (SPA, App)
- 외부 연계: CRM, 예약 시스템, 서비스 플랫폼 등
- 배포 환경: Cloud / On-Premise 협의
